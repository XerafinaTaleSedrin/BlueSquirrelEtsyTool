# Pursued Ideas Archive

This directory contains research for ideas that received a **Go** decision and were pursued.

---

## Purpose

When you decide to pursue an idea:
- The idea folder stays in the main repository initially
- As the idea becomes an active project, move the research folder here for reference
- Link to the actual project repository or workspace
- Track outcomes and learnings

---

## What Goes Here

**Move idea folders here when**:
- Idea has **[Go]** status
- You've started building/launching
- Research phase is complete
- Want to keep research accessible but separate from active evaluation

**Each folder should contain**:
- Original research files (brainstorm, market research, feasibility, decision framework)
- Link to active project (GitHub repo, product URL, etc.)
- Outcome tracking (what happened after pursuit)
- Lessons learned (what worked, what didn't)

---

## Folder Structure Example

```
pursued/
├── successful-idea-name/
│   ├── 01_brainstorm.md
│   ├── 02_market-research.md
│   ├── 03_feasibility-analysis.md
│   ├── 04_decision-framework.md
│   ├── OUTCOME.md (tracking results)
│   └── PROJECT_LINK.md (link to actual project)
└── another-pursued-idea/
    └── [same structure]
```

## Learning from Pursued Ideas

Periodically review pursued ideas to:
- Validate evaluation methodology (did scores predict success?)
- Learn which assumptions were accurate
- Understand what makes ideas work in practice
- Improve future evaluations

**Questions to ask**:
- Did market research accurately predict reality?
- Were feasibility estimates on target?
- Which dimension score was most predictive?
- What surprised you in execution?
- Would you still have pursued this with current knowledge?

---

## Updating Playbook

After pursuing ideas and seeing results, update:
- `playbook/patterns-that-work.md` - What made this idea succeed
- `playbook/red-flags.md` - If assumptions proved wrong, what should have warned you
- `playbook/evaluation-insights.md` - How to improve evaluation based on real outcomes

## Portfolio Link

Pursued ideas should remain linked in `ideas/00_IDEA_PORTFOLIO.md` with:

- Link to this archive folder
